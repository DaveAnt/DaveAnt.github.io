前言

每个语言都是从Hello World开始的,这篇文章就来Hello一下Shader,使用CG编写最简单的顶点/片元着色器,对UnityShader学习踏上第一步的征程。

1.最简单的着色器

Shader入门教程讲解了每个语义块的含义,直接在Pass语义块里开始编写CG,具体Shader代码如下所示：

Shader "Custom/SimpleShader"
{
    SubShader{
        Pass{
            CGPROGRAM

            #pragma vertex vert  
            #pragma fragment frag  
            #include "UnityCG.cginc"  

            float4 vert(float4 v : POSITION) : SV_POSITION{
                return UnityObjectToClipPos(v);
            }

            fixed4 frag() : SV_Target{
                return fixed4(1.0,1.0,1.0,1.0);
            }

            ENDCG
        }
    }
    FallBack "Diffuse"
}

首先需要保持良好的命名习惯,这样为材质指定Shader时就可以快速找到自定义的Shader了,然后可以看到Properties属性语义块不是必须的,在Pass里同样没有进行任何自定义的渲染设置或标签设置,最简单的Shader就一切从简吧！接着,就是由CGPROGRAM和ENDCG所包围的CG代码片段,首先最开始发现比较主要的三个编译指令：

    #pragma vertex vert  //vert指定的顶点函数名,随意命名
    #pragma fragment frag  //frag指定的片元函数名,随意命名
    #include "UnityCG.cginc"  //将UnityCG.cginc导入,其实不写也会默认导入的

顶点函数和片元函数基本都会命名成以上形式,这样命名很直观,pragma和include编译指令学过C语言都知道其实含义是类似的,UnityObjctToClipPos接口和下面语义是相同的。

    mul(UNITY_MATRIX_MVP,v);

对于这个变换矩阵不知道具体含义的,可以去看看本站的UnityShader运算,意思就是将模型空间上的顶点转换到剪裁空间的顶点,之后就是投影到二维的屏幕,当然不使用接口直接写成mul(UNITY_MATRIX_MVP,v)也是可以的,只不过5.6版本以上会强制转换成UnityObjectToClipPos,Unity还是很贴心的嘛。有些伙伴可能会有疑问：老子就不转换会怎么样？各位可以尝试一下return v会出现什么效果,圆形会变成椭圆形的(球体在摄像机剪裁空间之内,就会居中显示椭圆形)。 POSITION和SV_POSITION都是CG/HLSL中的语义,它们是不可省略,POSITION表示Unity输入是模型空间上的坐标,SV_POSITION表示Unity输出是剪裁空间中的坐标。接下来就是片元函数：

    fiexd4 frag() : SV_Target{
    	return fixed4(1.0,1.0,1.0,1.0);
    }

这个片元函数没有输入,它的输出是一个fixed4类型的变量,并且使用了SV_Target语义也是HLSL中的系统语义,它等同于告诉渲染器,把用户输出颜色存储到渲染目标(render target)中,这里将输出到默认的帧缓存中,片元着色器中的代码很简单,返回表示白色的fixed4类型变量,片元着色器输出的颜色每个分量范围在[0,1],其中(0,0,0)表示黑色,而(1,1,1)表示白色,可以对应调式这些值改变出任意颜色(r/256,b/256,g/256)。

2.模型数据来自？

从上面例子可以看到使用POSITION语义可以得到模型顶点位置,那么想要更多模型数据怎么办呢？比如想要得到模型上每个顶点的纹理坐标和法线方向。这个要求其实很常见的,使用纹理坐标来访问纹理,而法线可用于计算光照。因此需要为顶点着色器定义新的输入参数,这个输入参数不再是简单的数据类型,而是一个结构体。修改以后代码如下所示:

Shader "Custom/SimpleShader"
{
    SubShader{
        Pass{
            CGPROGRAM
            //使用vert函数和frag函数  
            #pragma vertex vert  
            #pragma fragment frag  
            #include "UnityCG.cginc"  

            //使用结构体来定义顶点着色器的输入
            struct a2v {
            // POSITION语义告诉Unity,用于模型空间的顶点坐标填充vertex变量
            float4 vertex : POSITION;
            // NORMAL语义告诉Unity,用模型空间按的法线方向填充normal变量
            float normal : NORMAL;
            //TEXCOORD0语义告诉Unity，用模型的第一套纹理坐标填充texcoord变量
            float4 texcoord : TEXCOORD0;
            };
            
            float4 vert(a2v v) : SV_POSITION{
                return UnityObjectToClipPos(v.vertex);
            }

            fixed4 frag() : SV_Target{
                return fixed4(1.0,1.0,1.0,1.0);
            }

            ENDCG
        }
    }
    FallBack "Diffuse"
}

在上面声明了新的结构体a2v,它包含了顶点着色器需要的模型数据。在a2v的定义中,用到了更多Unity支持的语义,如NORMAL和TEXCOORD0,当它们作为顶点着色器的输入时都有特定含义的,因为Unity会根据这些语义来填充这个结构体。对于顶点着色器的输出。Unity的支持的语义有:POSITION,TANGENT,NORMAL,TEXCOORD0,TEXCOORD1,TEXCOORD2,TEXCOORD3,COLOR等。为了创建自定义结构体,必须使用如下格式来定义：

struct StructName{
	Type Name ： Semantic;
	Tytpe Name ： Semantic;
}

命名成a2v,a表示应用,v表示顶点着色器,a2v就是从应用阶段传递到顶点着色器中,至于填充到POSITION、TANGENT、NORMAL这些语义中的数据究竟是从那里来的呢？在Unity中,它们是由使用材质的Mesh Render组件提供的。在每次调用DrawCall时,Mesh Render组件会把它负责渲染的模型数据发送给Unity Shader,模型通常包含一组三角面片,每个三角面片由3个顶点构成,而每个顶点又包含着了一些数据,例如顶点位置、法线、切线、纹理坐标、顶点颜色等,通过上面方法,就可以在顶点着色器中访问顶点的这些模型数据。

3.顶点着色器和片元着色器通信

在实践中,往往希望从顶点着色器输出一些数据到片元着色器,例如把模型的法线、纹理坐标等传递给片元着色器。这就涉及到顶点着色器和片元着色器之间的通信。为此需要定义出新的结构体。修改后的代码如下：

Shader "Custom/SimpleShader"
{
    SubShader{
        Pass{
            CGPROGRAM
            //使用vert函数和frag函数  
            #pragma vertex vert  
            #pragma fragment frag  
            #include "UnityCG.cginc"  

            //使用结构体来定义顶点着色器的输入
            struct a2v {
                // POSITION语义告诉Unity,用于模型空间的顶点坐标填充vertex变量
                float4 vertex : POSITION;
                // NORMAL语义告诉Unity,用模型空间按的法线方向填充normal变量
                float normal : NORMAL;
                //TEXCOORD0语义告诉Unity，用模型的第一套纹理坐标填充texcoord变量
                float4 texcoord : TEXCOORD0;
            };

            struct v2f {
                float4 pos : SV_POSITION;
                fixed3 color : COLOR0;
            };
            
            v2f vert(a2v v){
                v2f o;
                o.pos = UnityObjectToClipPos(v.vertex);
                o.color = v.normal * 0.5 + fixed3(0.9, 0.1, 0.5);
                return o;
            }

            fixed4 frag(v2f o) : SV_Target{
                return fixed4(o.color,0.8);
            }

            ENDCG
        }
    }
    FallBack "Diffuse"
}

这里定义的v2f表示用于在顶点着色器和片元着色器之间传递信息的,同样的,v2f中也需要指定每个变量的语义,在本例中(这里的v2f指定了SV_POSITION和COLOR0语义),在顶点着色器必须包含的语义是SV_POSITION,否则渲染器将无法得到剪裁空间中的顶点坐标,也无法把顶点坐标渲染到屏幕上。CLOLR0语义中的数据则可以由用户自行定义的,但一般都是存储颜色,例如逐顶点的漫反射颜色或逐顶点的高光反射颜色。类似语义还有COLOR1等。最后需要注意的是:顶点着色器是逐顶点调用,片元着色器是逐片元调用。片元着色器中的输入实际上是把顶点着色器的输出进行插值后得到的结果。

解决疑问

  • 模型空间直接投影为什么椭圆？
    直接使用模型空间进行屏幕投影,会发现任何模型都会居中变扁。这里分析一下为什么会出现这种效果,首先模型空间基本上是模型的重心坐标系,导致不进行转换的模型顶点必然是居中的,其实说白了这个效果其实就是模型放在相机近剪裁面的原点,然后横向拉伸。
  • 片元、顶点着色器关联方式
    了解渲染流水线的,都知道顶点着色器是在片元着色器之前的,所以顶点着色器返回的数据就是片元着色器可以获取到的数据(顶点着色器返回的必须包含剪裁后的坐标空间),顶点着色器返回数据以后,可以选择顶点着色器返回的数据输入,也可以选择不填写任何东西。
