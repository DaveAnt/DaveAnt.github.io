ScriptableProcessor

1.插件说明

你好！您是否遇到过这样的情况：项目中有一个名为DataTableHelperBase的抽象基类，其父类如TxtDataTableHelper、CsvDataTableHelper，XmlDataTableHelpers、JsonDataTableHelper等。并且您希望直接在检查器面板中动态地选择和设置父类的参数。 ScriptableProcessor就是解决上述问题的插件。它允许通过反射快速序列化和管理这些父类，并提供了一个用户友好的检查器面板以便于管理。 在设计游戏框架和插件时，它可以显著减少额外的MonoBehaviours的数量。ScrollCircleMaker就是基于这个插件开发的。

2.样式效果

视频预览：https://www.bilibili.com/video/BV1MH4y1G7Ts

在工程里有两个示例脚本：ExampleInspectorGUI和ExamplePropertyDrawer。具体初始化方法如下：

public class ExamplePropertyDrawer : MonoBehaviour
{
    [ScriptableProcessor]
    public ScriptableProcessor<TestScriptableHelperBase> ScriptabletHelper = new ScriptableProcessor<TestScriptableHelperBase>();

    void Start()
    {
        ScriptabletHelper[0].PrintTest();
        ScriptabletHelper[1].PrintTest();
    }
}

ScriptableProcessor＜TestScriptableHelperBase＞的最终监视器面板样式，如下图所示：

下拉选项是通过反射获得的，包括从TestScriptableHelperBase继承的所有父类。当创建新的父类时，开发人员不需要进行任何更改；检查器面板将自动显示新创建的父类作为选项。当选择父类时，可以为该父类序列化和编辑的属性将显示在红色框中。

3.微调监视面板

[ScriptableProcessor（isLock、isHeader、indentLevel）]三个参数的说明：

  • isLock 是否禁用添加和删除按钮，并且仅显示单个选择。
  • isHeader 是否删除标题而不显示折叠。
  • indentLevel 绘制ScriptableProcessor时的缩进。

4.注意事项

  • 如果需要反射其他程序集中的对象，请自己在AssemblyExt脚本中修改配置。
  • 如果基类不是ScriptableObject或MonoBehavior，则不会显示“自定义”选项。
