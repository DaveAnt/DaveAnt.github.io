前言

装饰器模式可以动态的给对象添加额外的职责,增加新的功能,思考一下和对象适配器的模式到底存在那些区别。

1.装饰器的使用

在蜘蛛侠的电影中,我们可以看到主角本来是一个普通人,当时被蜘蛛咬了以后,再加上他自己设计的道具,经过一番的装饰主角就变成了蜘蛛侠,我们可以用代码来模拟一下这个事情,首先定义普通人类,具体代码如下:

namespace AdapterPattern
{
    public interface IHero
    {
        void Atk();
        void Def();
        void Skill();
        void Look();
    }
 
    public class Person : IHero
    {
        public void Atk() {
            Console.WriteLine("普通攻击");
        }
 
        public void Def() {
            Console.WriteLine("普通的防御");
        }
 
        public void Skill() {
            Console.WriteLine("普通的技能");
        }
        
        public void Look(){
            Console.WriteLine("普通的外观");
        }
    }
}

普通人类大概就是这样的,但是蜘蛛侠就不一样了,他的各方面能力都出现了加强,这个时候我可以使用装饰器模式去创建出蜘蛛侠,具体代码如下:

using System;
using System.Collections.Generic;
using System.Text;
 
namespace AdapterPattern
{
    public class SpiderMan
    {
        private Person person;
 
        public SpiderMan(Person person)
        {
            this.person = person;
        }
 
        public void Atk()
        {
            Console.WriteLine("蜘蛛侠的攻击");
            person.Atk();
        }
 
        public void Def()
        {
            Console.WriteLine("蜘蛛侠的防御");
            person.Def();;
        }
 
        public void Skill()
        {
            Console.WriteLine("蜘蛛侠的技能");
            person.Skill();
        }
 
        public void Look()
        {
            Console.WriteLine("蜘蛛侠的外观");
            person.Look();
        }
    }
}

2.对象适配器和装饰器区别

首先需要知道对象适配器到底是何物？
相同点:对象适配器和装饰器实现的方式是一样的。
不同点:它们虽然都持有目标对象,适配器是对象不能满足新功能,但是新功能又不需要去重写的时候,主要是为了接口的适配。而装饰器目的是用新对象把旧对象包装起来,主要为了增强或者增加一些功能。

总结

1.装饰器的优缺点

优点:装饰类和被装饰类可以独立发展,不会相互耦合,装饰模式是继承的一个替代模式,装饰模式动态扩展实现类的功能。
缺点:如果多层装饰代码结构就会比较复杂。

2.装饰器的使用场景

1）当游戏人物有互通的逻辑的时候,把这些逻辑提取出来,然后创造不一样的游戏角色就使用适配器。
2）可以动态扩展类的功能。
