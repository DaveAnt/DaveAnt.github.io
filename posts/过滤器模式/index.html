前言

网上对过滤器的解释已经很多了,而且这个模式的概念理解起来不算很难,但是发现网上提供的Demo代码写得不好,所以这里就给大家介绍一下如何实现合理的过滤器模式。

1.网上资料的缺点

百度一下过滤器模式,基本上可以看到网上大部分资料,将过滤逻辑代码都写死在过滤器类里面,这说明了过滤器代码是不能重复利用的,这样真的比较糟糕,先给大家提供一下网上资料的代码,如下图所示: 这样应该怎么做呢？代码需要解耦,就需要把比较函数提取出来。经过思考,我们可以让过滤器类持有一个委托,这个委托就是用来过滤数据的,我们在构造函数中给过滤委托进行赋值即可。

2.可复用过滤器

复用过滤器的思想已经说清楚了,所以这里我们就直接尝试构建代码,先定义出抽象类,考虑一下过滤器基本的抽象函数和需要的实例有那些,具体代码如图下:

using System;
using System.Collections.Generic;
using System.Text;
 
namespace FilterPattern
{
    public abstract class IFilter
    {
        protected Func<object,object,bool> CompareFun;
 
        protected IFilter(Func<object, object, bool> CompareFun)
        {
            this.CompareFun = CompareFun;
        }
 
        public abstract List<T> ToFilter<T,T1>(List<T> dataSet, T1 condition);
    }
}

接下来需要继承这个抽象类,去具体实现这个过滤器类,具体代码如下:

using System;
using System.Collections;
using System.Collections.Generic;
using System.Text;
 
namespace FilterPattern
{
    public class FilterFactory : IFilter
    {
 
        public FilterFactory(Func<object, object, bool> CompareFun) : base(CompareFun)
        {
 
        }
 
        public override List<T> ToFilter<T, T1>(List<T> dataSet, T1 condition)    //单条件过滤
        {
            List<T> tmp = new List<T>();
            foreach (T data in dataSet)
            {
                if (CompareFun.Invoke(data, condition))
                    tmp.Add(data);
            }
            return tmp;
        }
 
        public List<T> ToMergeFilter<T, T1>(List<IFilter> filterSet, List<T> dataSet, T1 condition)  //组合过滤
        {
            List<T> tmp = dataSet;
            foreach (IFilter filter in filterSet)
            {
                tmp = filter.ToFilter(tmp, condition);
            }
            return tmp;
        }
 
    }
}

以上就把复用过滤器模式搞定了,自我感觉不错,代码也比较少。主要有两个功能（单个条件过滤和组合条件过滤）,但是还需要试一下到底会不会出现问题,首先我们需要模拟出一些数据再尝试进行过滤,具体代码如下:

List<Person> Persons = new List<Person>();
Persons.Add(new Person("Robert", 19, "Single"));
Persons.Add(new Person("John", 20, "Married"));
Persons.Add(new Person("Laura", 51, "Married"));
Persons.Add(new Person("Diana", 16, "Single"));
Persons.Add(new Person("Mike", 30, "Single"));
Persons.Add(new Person("Bobby", 9, "Single"));
//过滤器初始化
FilterFactory filter = new FilterFactory(delegate(object data, object condition)
{
    return ((Person) data).getAge() > (int)condition ;
});
List<Person> tmp = filter.ToFilter<Person, int>(Persons,18);

发现运行返回的列表,确实都是大于18岁的,唯一可惜的就是过滤函数用到了装箱和拆箱,性能上面有一定的消耗,至于如何组合条件过滤,我们把实例对象放到过滤器列表里,然后调用ToMergeFilter接口即可。

总结

1.以上过滤器的注意事项

1）以上的过滤器的实现没有考虑算法上面的优化,数据庞大时可能顶不住。
2）虽然把过滤函数提取出来,代码实现了解耦可复用,但是装箱和拆箱会有一定的消耗。

2.过滤器的使用场景

1）数据库的实现原理。
2）游戏中模糊查询其他玩家信息。
