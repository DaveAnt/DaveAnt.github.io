访问者模式

访问者模式

March 4, 2020 字数 2013 极速模式

前言

如果对象中的数据是稳定不变,但访问数据方式可能易变的或需扩展的。可以使用访问者模式,主要是为了对象中的数据和访问数据方式分离开来。

1.游戏分享链接

有天游戏策划跑过来和你说:嘿,伙计,把我们的游戏添加上分享功能模块吧,毕竟最好的推广方式是玩家与玩家之间的。比如需要时用户把游戏活动链接分享到微信可以获取一点奖励,然后我们的产品也更好的宣传开来了,首先需要先接入微信提供的工具开发包,然后就正常的思路编写代码即可,具体代码如下:

    public enum ShareType
    {
        Moments,
        Personal,
        GroupChat
    }
 
    public class ShareContent
    {       
        private string title;//标题
        private string content;//内容
        private string imageUrl;//图片链接
        private string iosLink;//游戏ios下载链接
        private string androidLink;//游戏ios下载链接
 
        public ShareContent(string title,string content,
                string imageUrl,string iosLink,string androidLink)
        {
            this.title = title;
            this.content = content;
            this.imageUrl = imageUrl;
            this.iosLink = iosLink;
            this.androidLink = androidLink;
        }
 
        public void ShareWeChat(bool isIos,ShareType shareType)
        {
            switch (shareType)
            {
                case ShareType.GroupChat:
                    if (isIos)
                        Console.WriteLine("苹果用户分享到微信朋友圈");
                    else
                        Console.WriteLine("安卓用户分享到微信朋友圈");
                    break;
                case ShareType.Personal:
                    if (isIos)
                        Console.WriteLine("苹果用户分享到微信个人");
                    else
                        Console.WriteLine("安卓用户分享到微信个人");
                    break;
                case ShareType.Moments:
                    if (isIos)
                        Console.WriteLine("苹果用户分享到微信群聊");
                    else
                        Console.WriteLine("安卓用户分享到微信群聊");
                    break;
            }
        }
    }

但是如果某天微信团队提供的工具开发包完全重构了,导致开发者也要将分享模块代码大量重构,但是对象中的数据不需要改变的,单单是调用时的一些逻辑需要调整,分享模块代码和数据又集中在一起,改起来可能比较吃力,所以为了以后方便突然醒悟还是换成访问者模式比较好,具体代码如下:

    public enum ShareType
    {
        Moments,
        Personal,
        GroupChat
    }
 
    public interface ICommonShare
    {
        void DoShare(ShareContent shareContent);
    }
 
    public class WeChatShareByIOS : ICommonShare
    {
        public void DoShare(ShareContent shareContent)
        {
            Console.WriteLine("苹果用户分享");
        }
    }
 
    public class WeChatShareByAndroid : ICommonShare
    {
        public void DoShare(ShareContent shareContent)
        {
            Console.WriteLine("安卓用户分享");
        }
    }
 
    public class ShareContent
    {       
        private string title;//标题
        private string content;//内容
        private string imageUrl;//图片链接
        private string iosLink;//游戏ios下载链接
        private string androidLink;//游戏ios下载链接
 
        public ShareContent(string title,string content,
                string imageUrl,string iosLink,string androidLink)
        {
            this.title = title;
            this.content = content;
            this.imageUrl = imageUrl;
            this.iosLink = iosLink;
            this.androidLink = androidLink;
        }
 
        public void DoShare(ICommonShare commonShare)
        {
            commonShare.DoShare(this);
        }
    }

访问者模式是将访问者对象传递给Content时,然后到Content内部以后将自身传递给访问者对象,具体调用代码如下:

shareContent.DoShare(new WeChatShareByAndroid());

2.分享样式可控怎么办？

不知你们再玩分享游戏或其他软件链接时,有没有遇到它们的分享样式时可控的,就是我们可以去掉标题、内容、图片,但是本质的软件下载链接是不可变的,这个时候怎么办呢？不要慌,具体代码如下:

    public interface ICommonShare
    {
        void DoShare(ShareContent shareContent);
        void DoShare(ShareTextContent shareContent);
        void DoShare(ShareImageContent shareContent);
    }
 
    public class WeChatShareByIOS : ICommonShare
    {
        public void DoShare(ShareContent shareContent)
        {
            Console.WriteLine("苹果用户分享");//全样式
        }
 
        public void DoShare(ShareTextContent shareContent)
        {
            Console.WriteLine("苹果用户分享");//文本样式
        }
 
        public void DoShare(ShareImageContent shareContent)
        {
            Console.WriteLine("苹果用户分享");//图片样式
        }
    }
 
    public class WeChatShareByAndroid : ICommonShare
    {
        public void DoShare(ShareContent shareContent)
        {
            Console.WriteLine("安卓用户分享");//全样式
        }
 
        public void DoShare(ShareTextContent shareContent)
        {
            Console.WriteLine("安卓用户分享");//文本样式
        }
 
        public void DoShare(ShareImageContent shareContent)
        {
            Console.WriteLine("安卓用户分享");//图片样式
        }
    }
 
    public abstract class CommonContent
    {
        protected string iosLink;//游戏ios下载链接
        protected string androidLink;//游戏ios下载链接
        public abstract void DoShare(ICommonShare commonShare);
    }
 
    public class ShareTextContent : CommonContent
    {
        private string title;//标题
        private string content;//内容
        public ShareTextContent(string title, string content,
                string imageUrl, string iosLink, string androidLink)
        {
            this.title = title;
            this.content = content;
            this.iosLink = iosLink;
            this.androidLink = androidLink;
        }
        public override void DoShare(ICommonShare commonShare)
        {
            commonShare.DoShare(this);
        }
    }
 
    public class ShareImageContent : CommonContent
    {
        private string imageUrl;//图片链接
        public ShareImageContent(string imageUrl, string iosLink, string androidLink)
        {
            this.imageUrl = imageUrl;
            this.iosLink = iosLink;
            this.androidLink = androidLink;
        }
        public override void DoShare(ICommonShare commonShare)
        {
            commonShare.DoShare(this);
        }
    }
 
    public class ShareContent : CommonContent
    {
        private string title;//标题
        private string content;//内容
        private string imageUrl;//图片链接
        public ShareContent(string title, string content,
                string imageUrl, string iosLink, string androidLink)
        {
            this.title = title;
            this.content = content;
            this.imageUrl = imageUrl;
            this.iosLink = iosLink;
            this.androidLink = androidLink;
        }
        public override void DoShare(ICommonShare commonShare)
        {
            commonShare.DoShare(this);
        }
    }

需要不同的样式就去创造不同数据对象,所以发现需要让对象中数据进行很多不相关的操作,而需要避免让这些操作"污染"这些对象的类,也不希望在增加新操作时修改这些类,用这个设计模式就很适合,虽然比较少用,用时确发挥了莫大的作用。

总结

1.访问者模式优缺点

优点: 1、符合单一职责原则。 2、优秀的扩展性、灵活性。
缺点:1、具体元素对访问者公布细节,违反了迪米特原则。 2、具体元素变更比较困难。 3、违反了依赖倒置原则,依赖了具体类,没有依赖抽象。

2.访问者模式使用场景

1.对象结构中对象对应的类很少改变,但经常需要在此对象结构上定义新的操作。
2.软件分享链接模块的实现。
