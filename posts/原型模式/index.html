原型模式

原型模式

March 19, 2020 字数 1161 极速模式

前言

原型模式是用于创建重复的对象,当类里有很多复杂参数的时候,我们可以避免使用拷贝(赋值)构造函数,不用自己去定义拷贝构造函数。

1.New与MemberwiseClone区别

using System;
namespace PrototypePattern
{
    public abstract class BasePrototype
    {
        public abstract BasePrototype Clone();
        public abstract BasePrototype NewClone();
    }
 
    public class Test: BasePrototype
    {
        private string name;
        private int num;
 
        public Test(Test test)
        {
            name = test.name;
            num = test.num;
        }
 
        public override BasePrototype Clone()
        {
            return (BasePrototype)MemberwiseClone();
        }
 
        public override BasePrototype NewClone()
        {
            return new Test(this);
        }
    }
 
}

我们知道原型模式主要是对象复制,至于用New还是用MemberwiseClone去实现原型模式呢？感受就是使用MemberwiseClone更加方便,C#的所有基类都是object,此函数就是基类里的。而使用New时我们还需要写一个拷贝构造函数,所以偷懒的话,咱们就可以使用MemberwiseClone,但是性能方面到底是使用New好还是C#提供给我们的MemberwiseClone好呢？这里我们来测试一下,代码而下:

public void TimeWatcher(BasePrototype basePrototype)
{
    int i;
    Stopwatch sw = new Stopwatch();
    sw.Start();
    for (i = 0; i < 1000000; i++)
    {
        basePrototype.NewClone();
    }
    sw.Stop();
    Console.WriteLine("NewClone用时:" + sw.Elapsed);
 
    sw.Reset();
    sw.Start();
    for (i = 0; i < 1000000; i++)
    {
        basePrototype.Clone();
    }
    sw.Stop();
    Console.WriteLine("Clone用时:" + sw.Elapsed);
}

运行代码以后,发现NewClone比Clone性能好很多,网上的一些资料说原型模式比New性能更快,个人感觉还是自己实践以后得到的结论才是靠谱的。所以当我们使用C#的时候,需要使用原型模式的时候,建议使用拷贝(复制)构造函数去实现这个设计模式,而且这样做比较的灵活,有些赋值不需要或者是需要加判断等等。

2.深拷贝和浅拷贝

MemberwiseClone函数的解释是浅复制,这样浅复制到底是什么呢？当我们的类里有引用类型时,使用此函数引用类型只会进行栈上的地址拷贝,所以当你复制出对象B（使用MemberwiseClone拷贝对象A）,修改对象B里的引用类型时,对象A也会改变。具体原因如下图所示: 这个就是浅复制（浅拷贝）,深拷贝就是在堆上都有副本了,深拷贝使用拷贝构造函数去实现,代码如下:

public class TwoNumber
{
    private int num1;
    private int num2;
}
public class Test: BasePrototype
{
    private string name;
    private int num;
    private TwoNumber two;

    public Test(Test test)
    {
        name = test.name;
        num = test.num;
        //深拷贝
        two = new TwoNumber();
        two.num1 = test.two.num1;
        two.num2 = test.two.num2;
        //浅拷贝
        //two = test.two;
    }

    public override BasePrototype NewClone()
    {
        return new Test(this);
    }
}

总结

原型模式的使用场景

1）游戏有相同的道具在地图显示时,我们就需要使用原型模式,浅复制的话,如果此道具被捡起来销毁以后,全部这个道具都会消失,所以我们需要真正的分身术,也就是深拷贝。
2）性能和安全有要求的场景。

原型模式的注意事项

1）.net环境下,发现MemberwiseClone比new性能慢,所以建议使用拷贝(复制)构造去实现原型模式。
2）当类的结构和属性很复杂的时候,想要简单和方便的实现原型模式,建议直接使用MemberwiseClone。
3）MemberwiseClone是浅副本,所以类中定义的变量有引用类型的时候,要小心！！！
