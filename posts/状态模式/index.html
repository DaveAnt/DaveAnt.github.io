前言

当类的某个属性发生改变时允许改变其内部的行为,可以减少分支语言和选择开关的集中。

1.Unity动画状态机

没有用过Unity动画状态机的小伙伴,这里先简单的讲解一下,就是可以通过可视化界面设置每个动画状态之间跳转的关系,可能用过的人感觉好神奇,但是Unity不是开源的,所以这里就尝试实现一下Untiy动画状态机模块,先给你们看看动画状态机的配置界面吧,让各位混个脸熟,具体如图所示: 可以通过连线来表达出状态之间切换的条件,先不考虑Unity是如何实现这个功能模块是可复用的,首先仅仅把以上这种情况模拟出来,具体代码如下:

    public enum ActionType
    {
        idle1,
        run,
        attack1,
        attack2,
        attack3,
    }
 
    public enum KeyType
    {
        key_j,
        key_k,
        key_l,
    }
 
    public class ActionPlayer
    {
        public ActionType actionType;
 
        public ActionPlayer(KeyType keyType)
        {
            actionType = ActionType.idle1;
        }
 
        void OnKeyboardClick(KeyType keyType)
        {
            switch (this.actionType)
            {
                case ActionType.run:
                    if (keyType == KeyType.key_j)
                    {
                        Console.WriteLine("执行攻击动作2");
                        this.actionType = ActionType.attack2;
                    }
                    break;
                case ActionType.attack1:
                    if (keyType == KeyType.key_k)
                    {
                        Console.WriteLine("执行攻击动作3");
                        this.actionType = ActionType.attack3;
                    }
                    else if (keyType == KeyType.key_j)
                    {
                        Console.WriteLine("执行攻击动作2");
                        this.actionType = ActionType.attack2;
                    }
                    break;
                case ActionType.attack2:
                    if (keyType == KeyType.key_k)
                    {
                        Console.WriteLine("执行攻击动作3");
                        this.actionType = ActionType.attack3;
                    }
                    break;
                case ActionType.attack3:
                    if (keyType == KeyType.key_l)
                    {
                        Console.WriteLine("执行待机动画");
                        this.actionType = ActionType.idle1;
                    }
                    break;
                case ActionType.idle1:
                    if (keyType == KeyType.key_l)
                    {
                        Console.WriteLine("执行跑步动画");
                        this.actionType = ActionType.run;
                    }
                    else if (keyType == KeyType.key_j)
                    {
                        Console.WriteLine("执行攻击动作1");
                        this.actionType = ActionType.attack1;
                    }
                    break;
            }
        }
    }

代码基本把上述图片中的情况算是实现了,但是分支语句过多,可能再复杂点的动画状态机就会有更多的分支语句和选择开关的嵌套,当你实在忍受不了的时候,今天主角将很好的帮你解决这个问题,使代码逻辑更加清晰,用状态模式调整代码如下:

    public interface IAction
    {
        void doAction(ActionPlayer actionPlayer,KeyType keyType);
    }
 
    public class IdleAction : IAction
    {
        public void doAction(ActionPlayer actionPlayer, KeyType keyType)
        {
            if (keyType == KeyType.key_l)
            {
                Console.WriteLine("执行跑步动画");
                actionPlayer.SetAction(ActionType.run,new RunAction());
            }
            else if (keyType == KeyType.key_j)
            {
                Console.WriteLine("执行攻击动作1");
                actionPlayer.SetAction(ActionType.attack1, new AtkAction1());
            }
        }
    }
 
    public class RunAction : IAction
    {
        public void doAction(ActionPlayer actionPlayer, KeyType keyType)
        {
            if (keyType == KeyType.key_l)
            {
                Console.WriteLine("执行跑步动画");
                actionPlayer.SetAction(ActionType.run, new RunAction());
            }
            else if (keyType == KeyType.key_j)
            {
                Console.WriteLine("执行攻击动作1");
                actionPlayer.SetAction(ActionType.attack1, new AtkAction1());
            }
        }
    }
 
    public class AtkAction1 : IAction
    {
        public void doAction(ActionPlayer actionPlayer, KeyType keyType)
        {
            if (keyType == KeyType.key_k)
            {
                Console.WriteLine("执行攻击动作3");
                actionPlayer.SetAction(ActionType.attack3, new AtkAction3());
            }
            else if (keyType == KeyType.key_j)
            {
                Console.WriteLine("执行攻击动作2");
                actionPlayer.SetAction(ActionType.attack2, new AtkAction2());
            }
        }
    }
 
    public class AtkAction2 : IAction
    {
        public void doAction(ActionPlayer actionPlayer, KeyType keyType)
        {
            if (keyType == KeyType.key_k)
            {
                Console.WriteLine("执行攻击动作3");
                actionPlayer.SetAction(ActionType.attack3, new AtkAction3());
            }
        }
    }
 
    public class AtkAction3 : IAction
    {

        {
            if (keyType == KeyType.key_l)
            {
                Console.WriteLine("执行待机动画");
                actionPlayer.SetAction(ActionType.idle1, new IdleAction());

    public class ActionPlayer
    {
        private ActionType actionType;
        private IAction action;
 
        public ActionPlayer(KeyType keyType)
        {
            actionType = ActionType.idle1;
            action = new IdleAction();
        }
 
        public ActionType GetType()
        {
            return actionType;
        }
 
        public void SetAction(ActionType actionType,IAction action)
        {
            this.actionType = actionType;
            this.action = action;
        }
 
        public void DoAction(KeyType keyType)
        {
            action.doAction(this, keyType);
        }
    }

使用状态模式会明显的发现减少了动作状态切换的分支判断的逻辑代码,只不过每个状态都将对应着一个状态类。对象的行为依赖于它的状态（属性）,会根据一定条件先改变状态而改变其行为。

2.优化状态模式的代码

状态行为改变时,最好不要反复的new行为,这样会增加GC的压力,如图所示: 直接让目标对象持有每个行为,在构造函数里面给它初始化一次即可,这样就不用切换动作状态时就去new一次,建议各位在编程时要注意这方面的问题,没有必要的情况下频繁的去new对象,虽然不用你去释放垃圾或者工程小时感觉不出来明显的问题,还是要养成良好的习惯,具体代码如下:

    {
        idle1,
        run,
        attack1,
        attack2,
        attack3,

    public enum KeyType

        key_l,

            }

            {
                Console.WriteLine("执行攻击动作1");
                actionPlayer.SetAction(ActionType.attack1);
            }
        }

    public class AtkAction2 : IAction
    {
        public void doAction(ActionPlayer actionPlayer, KeyType keyType)
        {
            if (keyType == KeyType.key_k)
            {
                Console.WriteLine("执行攻击动作3");
                actionPlayer.SetAction(ActionType.attack3);
            }
        }
    }
 
    public class AtkAction3 : IAction
    {
        public void doAction(ActionPlayer actionPlayer, KeyType keyType)
        {
            if (keyType == KeyType.key_l)
            {
                Console.WriteLine("执行待机动画");
                actionPlayer.SetAction(ActionType.idle1);
            }
        }
    }
 
    public class ActionPlayer
    {
        private ActionType actionType;
        private IAction action;
        private IAction idle, run, atk1, atk2, atk3;
 
        public ActionPlayer(KeyType keyType)
        {
            actionType = ActionType.idle1;
         
            idle = new IdleAction();
            run = new RunAction();
            atk1 = new AtkAction1();
            atk2 = new AtkAction2();
            atk3 = new AtkAction3();
 
            action = idle;
        }
 
        public ActionType GetType()
        {
            return actionType;
        }
 
        public void SetAction(ActionType actionType)
        {
            this.actionType = actionType;
            switch(actionType)
            {
                case ActionType.attack1:
                    action = atk1;
                    break;
                case ActionType.attack2:
                    action = atk2;
                    break;
                case ActionType.attack3:
                    action = atk3;
                    break;
                case ActionType.idle1:
                    action = idle;
                    break;
                case ActionType.run:
                    action = run;
                    break;
            }
        }
 
        public void DoAction(KeyType keyType)
        {
            action.doAction(this, keyType);
        }
    }

3.动画状态机的限制

小节三讲述的东西和状态模式已经没有丝毫关系了,所以对于Unity的底层原理不敢兴趣的小伙伴可以绕道而行了,假设上面的这些动画都对应着一个枚举,具体代码如下:

    public enum ActionType : byte
    {
        idle1,
        run,
        attack1,
        attack2,
        attack3,
    }

这样应该如何去表现这些枚举的连线关系呢？其实它们之间的连线关系也是可以用枚举表示的,具体代码如下:

    public enum LinkType : byte
    {
        link1 = ActionType.idle1 | ActionType.run | 1 << 7,//表示低位指向高位(idle-->run)
        link2 = ActionType.idle1 | ActionType.run,//表示高位指向低位(run->idle)
        link3 = ActionType.idle1 | ActionType.attack1 | 1 << 7,
        link4 = ActionType.idle1 | ActionType.attack1,
        link5 = ActionType.idle1 | ActionType.attack3 | 1 << 7,
        link7 = ActionType.run | ActionType.attack2 | 1 << 7,
        link8 = ActionType.attack1 | ActionType.attack2 | 1 << 7,
        link9 = ActionType.attack1 | ActionType.attack3 | 1 << 7,
        link10 = ActionType.attack2 | ActionType.attack3 | 1 << 7,
    }

最上面给出的动画状态机连线图完全可以靠以上枚举代码表示,其实Unity再编辑连线时可能生成类似与上面枚举类似的数据,现在已经可以表示连线关系了,接下来考虑一下动画切换函数SetTrigger是如何实现的,具体代码可能如下:

    public void SetTrigger(ActionType doType)
    {
        if (this.actionType > doType &&
            Enum.IsDefined(typeof(LinkType), this.actionType | doType))
        {
            Console.WriteLine("执行动作"+doType);
        }
        else if (this.actionType < doType &&
            Enum.IsDefined(typeof(LinkType), this.actionType | doType | (ActionType)0b10000000))
        {
            Console.WriteLine("执行动作" + doType);
        }
        else
        {
            Console.WriteLine("不存在指向关系" +this.actionType + "->" + doType);
        }
    }

虽然SetTrigger需要传递的参数是字符串,这里直接传递了枚举,其实差不多啦,枚举和字符串是可以相互转换。这里只是思考一下动画状态机模块可能实现的原理,具体Unity是不是这样的,我也母鸡啊~（在下的脑补而已）。

总结

1.状态模式优缺点:

优点:1、封装了转换规则。 2、代码观赏度提升,结构也变的比较清晰,减少过多的分支语句 3、将所有与某个状态有关的行为放到一个类中,并且方便增加新的状态。
缺点:1、状态模式的使用导致过多的子类。 2、状态模式的结构与实现都较为复杂,代码排错会比较难。

2.状态模式使用场景:

1.Unity里的动画状态机。
2.选择开关、分支语句的代替者。
