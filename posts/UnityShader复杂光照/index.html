UnityShader复杂光照

UnityShader复杂光照

May 2, 2020 字数 2784 极速模式

前言

基本光照篇中讲述了只适合场景中存在一个平行光的渲染,很明显单独掌握这点知识完全不够用的,所以需要掌握更复杂的光照渲染。

1.光源类型

光源属性有光源的位置、方向(到某点的方向)、颜色、强度以及衰减(到某点的衰减,与该点到光源距离有关)五个属性,这样不同的光源类型会给Shader碰撞出什么不一样的火花呢？接下里就简单讲述每个光源的概念。

  • 平行光
    它的几何定义最简单,也是Unity最常用的光源类型,它可以照亮的范围时没有限制,在引擎中通常扮演太阳这样的角色。它是唯一没有位置概念的光源,由于没有位置的概念所有也没有衰减的概念,它的几何属性只有方向。
  • 点光源
    它照亮空间是定义成球体的,由点光源中心向所有方向延伸的光,球体半径可以通过Range属性来调整。所以点光源是存在位置和衰减的概念,点光源中心光照强度最大,球体边缘为0,衰减值可以由有关函数定义,它可以扮演火柴堆光源等等。
  • 聚光灯
    聚光灯是光源中最复杂的一种,它的照亮空间是定义成锥形区域的,通过Range属性决定锥形区域半径,而锥体的张开角度由Spot Angle属性决定。在锥体顶点处光照强度最强,在锥形边界强度为0,其中衰减值可以由一个函数定义,这个函数相对于点光源衰减计算公式更加复杂,需要判断点是否在锥形范围内,它可以扮演手电筒或路灯光源等等。
  • 区域光源
    这个光源性能是最高的,因为区域光在“实时光照”模式下是无效的,仅在“烘焙光照”模式下有用。区域光源可以设置作用区域样式,区域光会均匀的照亮作用区域,虽然区域光没有范围属性可以调整,但是光强度也是会随着距离光源越远而递减,它可以扮演手摄影用的柔光灯。

2.实战渲染不同光源

接下来就来看看UnityShader中如何它们五个属性：位置、方向、颜色、强度以及衰减,使用前向渲染来实现平行光和点光源共同照亮物体。

Shader "Unity Shaders Book/Chapter 9/Forward Rendering" {
	Properties {
		_Diffuse ("Diffuse", Color) = (1, 1, 1, 1)
		_Specular ("Specular", Color) = (1, 1, 1, 1)
		_Gloss ("Gloss", Range(8.0, 256)) = 20
	}
	SubShader {
		Tags { "RenderType"="Opaque" }
		
		Pass {
			// Pass for ambient light & first pixel light (directional light)
			Tags { "LightMode"="ForwardBase" }
		
			CGPROGRAM
			
			// Apparently need to add this declaration 
			#pragma multi_compile_fwdbase	
			
			#pragma vertex vert
			#pragma fragment frag
			
			#include "Lighting.cginc"
			
			fixed4 _Diffuse;
			fixed4 _Specular;
			float _Gloss;
			
			struct a2v {
				float4 vertex : POSITION;
				float3 normal : NORMAL;
			};
			
			struct v2f {
				float4 pos : SV_POSITION;
				float3 worldNormal : TEXCOORD0;
				float3 worldPos : TEXCOORD1;
			};
			
			v2f vert(a2v v) {
				v2f o;
				o.pos = UnityObjectToClipPos(v.vertex);
				
				o.worldNormal = UnityObjectToWorldNormal(v.normal);
				
				o.worldPos = mul(unity_ObjectToWorld, v.vertex).xyz;
				
				return o;
			}
			
			fixed4 frag(v2f i) : SV_Target {
				fixed3 worldNormal = normalize(i.worldNormal);
				fixed3 worldLightDir = normalize(_WorldSpaceLightPos0.xyz);
				
				fixed3 ambient = UNITY_LIGHTMODEL_AMBIENT.xyz;
				
			 	fixed3 diffuse = _LightColor0.rgb * _Diffuse.rgb * max(0, dot(worldNormal, worldLightDir));

			 	fixed3 viewDir = normalize(_WorldSpaceCameraPos.xyz - i.worldPos.xyz);
			 	fixed3 halfDir = normalize(worldLightDir + viewDir);
			 	fixed3 specular = _LightColor0.rgb * _Specular.rgb * pow(max(0, dot(worldNormal, halfDir)), _Gloss);

				fixed atten = 1.0;
				
				return fixed4(ambient + (diffuse + specular) * atten, 1.0);
			}
			
			ENDCG
		}
	
		Pass {
			// Pass for other pixel lights
			Tags { "LightMode"="ForwardAdd" }
			
			Blend One One
		
			CGPROGRAM
			
			// Apparently need to add this declaration
			#pragma multi_compile_fwdadd
			
			#pragma vertex vert
			#pragma fragment frag
			
			#include "Lighting.cginc"
			#include "AutoLight.cginc"
			
			fixed4 _Diffuse;
			fixed4 _Specular;
			float _Gloss;
			
			struct a2v {
				float4 vertex : POSITION;
				float3 normal : NORMAL;
			};
			
			struct v2f {
				float4 pos : SV_POSITION;
				float3 worldNormal : TEXCOORD0;
				float3 worldPos : TEXCOORD1;
			};
			
			v2f vert(a2v v) {
				v2f o;
				o.pos = UnityObjectToClipPos(v.vertex);
				
				o.worldNormal = UnityObjectToWorldNormal(v.normal);
				
				o.worldPos = mul(unity_ObjectToWorld, v.vertex).xyz;
				
				return o;
			}
			
			fixed4 frag(v2f i) : SV_Target {
				fixed3 worldNormal = normalize(i.worldNormal);
				#ifdef USING_DIRECTIONAL_LIGHT
					fixed3 worldLightDir = normalize(_WorldSpaceLightPos0.xyz);
				#else
					fixed3 worldLightDir = normalize(_WorldSpaceLightPos0.xyz - i.worldPos.xyz);
				#endif
				
				fixed3 diffuse = _LightColor0.rgb * _Diffuse.rgb * max(0, dot(worldNormal, worldLightDir));
				
				fixed3 viewDir = normalize(_WorldSpaceCameraPos.xyz - i.worldPos.xyz);
				fixed3 halfDir = normalize(worldLightDir + viewDir);
				fixed3 specular = _LightColor0.rgb * _Specular.rgb * pow(max(0, dot(worldNormal, halfDir)), _Gloss);
				
				#ifdef USING_DIRECTIONAL_LIGHT
					fixed atten = 1.0;
				#else
					#if defined (POINT)
				        float3 lightCoord = mul(unity_WorldToLight, float4(i.worldPos, 1)).xyz;
				        fixed atten = tex2D(_LightTexture0, dot(lightCoord, lightCoord).rr).UNITY_ATTEN_CHANNEL;
				    #elif defined (SPOT)
				        float4 lightCoord = mul(unity_WorldToLight, float4(i.worldPos, 1));
				        fixed atten = (lightCoord.z > 0) * tex2D(_LightTexture0, lightCoord.xy / lightCoord.w + 0.5).w * tex2D(_LightTextureB0, dot(lightCoord, lightCoord).rr).UNITY_ATTEN_CHANNEL;
				    #else
				        fixed atten = 1.0;
				    #endif
				#endif

				return fixed4((diffuse + specular) * atten, 1.0);
			}
			
			ENDCG
		}
	}
	FallBack "Specular"
}

第一个Pass使用Base Pass,并且为了可以获取到正确的光照变量,需要使用了#pragma multi_complie_fwdbase指令。在Base Pass中处理场景中的环境光和平行光等等,如果场景存在多个平行光,Unity会选择最亮平行光进行逐像素处理,其他会按照逐顶点或Additional Pass中按逐像素方式处理,不存在光源就当全黑处理。,可以通过_WorldSpaceLightPos0来得到平行光方向,_LightColor0来得到它的颜色和强度(它已经是颜色和强度相乘结果)。

//计算环境光
fixed3 ambient = UNITY_LIGHTMODEL_AMBIENT.xyz;
//计算漫反射
fixed3 diffuse = _LightColor0.rgb*_Diffuse.rgb*max(0,dot(worldNormal,worldLightDir));
...
//计算高光
fixed3 specular = _LightColor0.rgb*_Specular.rgb*pow(max(0,dot(worldNormal,halfDir)),_Gloss);
//衰减因子
fixed3 atten = 1.0;
return fixed3(ambient + (diffuse+specular)*atten,1.0);

接下来就是为场景中其他逐像素光源定义Additional Pass,建议先去看看渲染路径,这里的混合系数使用的Blend One One不是必须的,常见还可用Blend SrcAlpha One。接下来查看如何计算不同光源方向：

#ifdef USING_DIRECTIONAL_LIGHT
	fixed3 worldLightDir = normalize(_WorldSpaceLightPos0.xyz);
#else
	fixed3 worldLightDir = normalize(_WorldSpaceLightPos0.xyz - i.worldPosition.xyz);
#endif

光源方向可以直接由_WorldSpaceLightPos0.xyz,这值在平行光类型是方向,其他光源表示世界空间下的光源位置,需要得到光源方向的话,需要这个位置减去世界空间下的顶点位置。最后就是处理不同光源的衰减。

#ifdef USING_DIRECTIONAL_LIGHT
	fixed atten = 1.0;
#else
	float3 lightCoord = mul(_LightMatrix0,float4(i.worldPosition,1)).xyz;
	fixed atten = tex2D(_LightTexture0,dot(lightCoord,lightCoord).rr).UNITY_ATTEN_CHANNEL;

可以通过USING_DIRECTIONAL_LIGHT来决定处理光源类型。

3.Unity的光照衰减

在Unity中可以使用衰减纹理而避免依赖数学公式的复杂性,只需要对纹理进行采样即可,这个是典型的空间换时间的策略,但是会存在一些弊端：需要预处理得到采用纹理,而纹理大小也会影响衰减精度;不直观给,同时也不方便,把数据存储到查找表中,就无法使用其他数学公式来计算衰减。
Unity默认就是采用衰减纹理(_LightTexture0)来提高点光源、聚光源的性能,如果对该光源使用了cookie那么衰减纹理就是_LightTextureB0,通常只关心_LightTexture0对角线上的纹理颜色值,这些值表明了在光源空间中不同位置的点衰减值。比如(0,0)点表明与光源位置重合点的衰减值,而(1,1)点表明在光源空间中所关心的距离最远点的衰减值。
为了对_LightTexture0纹理采样得到给定点到该光源的衰减值,首先需要得到该点在光源空间中的位置,这是通过_LightMatrix0变换矩阵得到的,接下来只需要把_LightMatrix0和世界空间中的顶点坐标相乘即可得到光源空间中的相应位置。

float3 lightCoord = mul(_LightMatrix0,float4(i,_worldPosition,1)).xyz;

然后可以使用坐标模的平方对衰减纹理进行采样得到衰减值。

fixed atten = tex2D(_LightTexture0,dot(lightCoord,lightCoord).rr).UNITY_ATTEN_CHANNEL;

使用光源空间中顶点距离的平方进行纹理采样,之所以没有使用距离值来采样是因为这种方法可以避免开方操作。然后使用宏定义UNITY_ATTEN_CHANNEL来得到衰减纹理中的衰减值所在的分量,以得到最终的衰减值。尽管纹理采样方式可以减少计算衰减的复杂度,但有时还是希望可以在代码中利用公式来计算光源的衰减,接下来就是计算衰减的公式。

float distance = length(_WorldSpaceLightPos0.xyz - i.worldPosition.xyz);
atten = 1.0 / distance;

Unity文档没有给出内置衰减计算的公式,而且UnityShader无法得到光源范围、聚光灯朝向、张开角度等信息,因此无法得到满意的效果,只期待未来UnityShader可以开发更多变量给开发者使用。

扩展知识

创建光源默认情况下的Render Mode是Auto,这意味着Unity会在背后判断那些光源会按逐像素处理,而哪些按逐顶点或SH方式处理。默认情况下物体接受除最亮的平行光外的四个逐像素光照(在Edit/Project Settings/Quality/Pixel Light Count设置逐像素数目)。
在渲染事件中,Unity会首先清除颜色、深度和模板缓冲,为后面渲染做准备,然后依次执行Pass将光源应用到物体上得到最终的渲染结果(可以打开Unity的帧调式器查看渲染事件),Unity会按照某个规则对其进行重要程度的排序(比如光源远近),当把场景中的光源全部设置成Not Important,物体就仅仅会受到环境光的影响。
