UnityShader透明效果

UnityShader透明效果

April 30, 2020 字数 2781 极速模式

前言

透明是游戏中经常要使用到的一种效果。在实际渲染中要实现透明效果,通常会在渲染模型时控制它的透明通道(Alpha Channel)。

1.基本概念

在Unity中通常使用两种方式来实现透明效果:第一种使用透明度测试(Alpha Test),这种方式其实无法得到真正的半透明效果;另外一种就是透明度混合(Alpha Blending),开启透明度混合后,每个片元除了颜色值和深度值之外,还有一个范围[0,1]的透明度属性。
在实现透明效果前,先来讲述渲染顺序的物体,通常情况并不需要关心模型A、B、C的顺序,由于强大的深度缓冲(depth buffer,也称为z-buffer)技术的存在。它用于解决可见性问题,用来决定模型哪个渲染在最前面,而哪些又是被遮挡的。基本的思想就是:根据深度缓冲中的值进行比较,如果值距离摄像机比较远,说明被遮挡应该被舍弃掉,否则片源应该覆盖掉原来区域上的像素值。

  • 透明度测试
    它采用“极端霸道”的机制,只要片元的透明度不满足条件,那它对应的片元就会被舍弃。舍弃的片元将不再进行任何处理,不然的话就是按照不透明模型处理方式处理,进行深度测试、深度写入等。这样说明不必关闭升读缓冲技术,要不完全透明,要不完全不透明。
  • 透明度混合
    这种方式可以得到真正半透明效果,它会使用当前片元透明度作为混合因子,与已经存储的颜色缓冲中的颜色值进行混合得到新颜色。透明度混合需要关闭深度写入,这样导致要十分小心物体渲染顺序。需要注意透明度混合只关闭深度写入,但没有关闭深度测试,还是会比较它的深度值进行剔除,所以应该先渲染不透明物体再是透明物体。

这样UnityShader这么控制渲染顺序呢,Unity为解决渲染顺序问题提供了渲染队列(render queue),可以使用SubShader的Queue标签来决定模型将归于哪个渲染队列。Unity内部使用一系列整数索引表示每个渲染队列,且索引号越小表示越早被渲染,接下来给出表格。

名称 队列索引号 描述
Background 1000 会在任何其他队列之前渲染,通常使用该队列来渲染绘制在背景上的物体。
Geometry 2000 默认渲染队列,不透明物体使用这个队列。
AlphaTest 2450 需要透明测试的物体使用的队列,从Geometry队列分开是为了更加高效。
Transparent 3000 任何要使用透明度混合的物体需要使用该队列,注意需要关闭深度写入。
Overlay 4000 该队列用于叠加效果,任何在最后渲染物体都应该使用这个物体。

所以想要开启透明度混合必须存在以下代码端：

SubShader{
	Tags{"Queue"="Transparent"}
	Pass{
		ZWrite Off//关闭深度测试
		Blend SrcAlpha OneMinusSrcAlpha//开启Blend命令
		...
	}
}

2.实战透明度效果

  • 透明度测试
    片元透明度不满足条件(通常小于某个阀值),那么它对应的片元就会舍弃。被舍弃的片元将不会再进行任何处理,也不会对颜色缓冲产生任何影响,否则就会按照普通的不透明物体方式处理它。
Shader "Custom/Alpha Test" {
	Properties {
		_Color ("Color Tint", Color) = (1, 1, 1, 1)
		_MainTex ("Main Tex", 2D) = "white" {}
		_Cutoff ("Alpha Cutoff", Range(0, 1)) = 0.5
	}
	SubShader {
		Tags {"Queue"="AlphaTest" "IgnoreProjector"="True" "RenderType"="TransparentCutout"}
		
		Pass {
			Tags { "LightMode"="ForwardBase" }
			
			CGPROGRAM
			
			#pragma vertex vert
			#pragma fragment frag
			
			#include "Lighting.cginc"
			
			fixed4 _Color;
			sampler2D _MainTex;
			float4 _MainTex_ST;
			fixed _Cutoff;
			
			struct a2v {
				float4 vertex : POSITION;
				float3 normal : NORMAL;
				float4 texcoord : TEXCOORD0;
			};
			
			struct v2f {
				float4 pos : SV_POSITION;
				float3 worldNormal : TEXCOORD0;
				float3 worldPos : TEXCOORD1;
				float2 uv : TEXCOORD2;
			};
			
			v2f vert(a2v v) {
				v2f o;
				o.pos = UnityObjectToClipPos(v.vertex);
				
				o.worldNormal = UnityObjectToWorldNormal(v.normal);
				
				o.worldPos = mul(unity_ObjectToWorld, v.vertex).xyz;
				
				o.uv = TRANSFORM_TEX(v.texcoord, _MainTex);
				
				return o;
			}
			
			fixed4 frag(v2f i) : SV_Target {
				fixed3 worldNormal = normalize(i.worldNormal);
				fixed3 worldLightDir = normalize(UnityWorldSpaceLightDir(i.worldPos));
				
				fixed4 texColor = tex2D(_MainTex, i.uv);
				
				// Alpha test
				clip (texColor.a - _Cutoff);
				// Equal to 
//				if ((texColor.a - _Cutoff) < 0.0) {
//					discard;
//				}
				
				fixed3 albedo = texColor.rgb * _Color.rgb;
				
				fixed3 ambient = UNITY_LIGHTMODEL_AMBIENT.xyz * albedo;
				
				fixed3 diffuse = _LightColor0.rgb * albedo * max(0, dot(worldNormal, worldLightDir));
				
				return fixed4(ambient + diffuse, 1.0);
			}
			
			ENDCG
		}
	} 
	FallBack "Transparent/Cutout/VertexLit"
}

在Properties语义中声明了范围[0,1]之间属性_Cutoff(因为纹理像素的透明度就是在此范围),它用来决定调用clip就行透明度测试时使用到的判断条件。
在Tags标签把Queue标签设置为AlphaTest;RenderType标签可以把Shader归入到提前定义的指定组中;IgnoreProjector设置为true,意味着不会收到投影器(Projectors)的影响。
在顶点着色器里这种代码已经出现很多次,在顶点着色器计算出世间空间的法线方向和顶点位置以及变换后的纹理坐标再传到片元着色器中。
片元着色器中的clip函数定义中,如果texColor.a - Cutoff是为负数就会被完全舍弃掉,这种处理方式很少在游戏中用到,但是性能效果好。

  • 透明度混合
    透明度混合实现比透明度测试更复杂、渲染代价更大,透明度混合可以得到真正的半透明效果。它会使用当前片元的透明度作为混合因子,与已经存储颜色缓冲中的颜色进行混合得到新颜色,但透明度混合必须小心物体渲染顺序。
    为了进行混合,需要使用Unity提供的内置混合命令(Blend)。需要实现半透明的效果就需要把当前自身颜色和已经存在与颜色缓冲中的颜色进行混合,混合时使用函数就由该指令决定的,下面是Blend命令相关的语义。
语义 描述
Blend Off 关闭混合
Blend SrcFactor DstFactor 开启混合并设置混合因子。源颜色(片元生产的色块)乘以SrcFactor,而目标颜色(已存在缓冲中的颜色)也会乘以DstFactor,最后两者相加刷新颜色缓冲。
Blend SrcFactor DstFactor SrcFactorA DstFactorA 和上面几乎相同,只是使用不同因子来混合透明通道。
BlendOp BlendOperation 并非是把源颜色和目标颜色简单相加后混合,而使用BlendOperation对它们进行其他操作。

透明度混合必须要在Pass中使用Blend命令,设置混合因子和打开混合模式。这里会把源颜色的混合因子(SrcFactor)设为SrcAlpha,而目标颜色的混合因子DstFactor设为OneMinusSrcAlpha,这意味着经过混合新颜色公式是：DstColornew=SrcAlpha*SrcColor+(1-SrcAlpha)*DstColorold

Shader "Custom/Alpha Blend" {
	Properties {
		_Color ("Color Tint", Color) = (1, 1, 1, 1)
		_MainTex ("Main Tex", 2D) = "white" {}
		_AlphaScale ("Alpha Scale", Range(0, 1)) = 1
	}
	SubShader {
		Tags {"Queue"="Transparent" "IgnoreProjector"="True" "RenderType"="Transparent"}
		
		Pass {
			Tags { "LightMode"="ForwardBase" }

			ZWrite Off
			Blend SrcAlpha OneMinusSrcAlpha
			
			CGPROGRAM
			
			#pragma vertex vert
			#pragma fragment frag
			
			#include "Lighting.cginc"
			
			fixed4 _Color;
			sampler2D _MainTex;
			float4 _MainTex_ST;
			fixed _AlphaScale;
			
			struct a2v {
				float4 vertex : POSITION;
				float3 normal : NORMAL;
				float4 texcoord : TEXCOORD0;
			};
			
			struct v2f {
				float4 pos : SV_POSITION;
				float3 worldNormal : TEXCOORD0;
				float3 worldPos : TEXCOORD1;
				float2 uv : TEXCOORD2;
			};
			
			v2f vert(a2v v) {
				v2f o;
				o.pos = UnityObjectToClipPos(v.vertex);
				
				o.worldNormal = UnityObjectToWorldNormal(v.normal);
				
				o.worldPos = mul(unity_ObjectToWorld, v.vertex).xyz;
				
				o.uv = TRANSFORM_TEX(v.texcoord, _MainTex);
				
				return o;
			}
			
			fixed4 frag(v2f i) : SV_Target {
				fixed3 worldNormal = normalize(i.worldNormal);
				fixed3 worldLightDir = normalize(UnityWorldSpaceLightDir(i.worldPos));
				
				fixed4 texColor = tex2D(_MainTex, i.uv);
				
				fixed3 albedo = texColor.rgb * _Color.rgb;
				
				fixed3 ambient = UNITY_LIGHTMODEL_AMBIENT.xyz * albedo;
				
				fixed3 diffuse = _LightColor0.rgb * albedo * max(0, dot(worldNormal, worldLightDir));
				
				return fixed4(ambient + diffuse, texColor.a * _AlphaScale);
			}
			
			ENDCG
		}
	} 
	FallBack "Transparent/VertexLit"
}

在Properties中添加了_AlphaScale用于透明纹理的基础上整体的透明度。并设置了片元着色器返回值中的透明通道,它是纹理像素的透明通道和材质参数_AlphaScale的乘积。

3.扩展知识

  • 透明测试的用处
    是否存在疑问,为什么需要透明测试？透明测试只有完全透明或不透明,直接去掉物体勾选或子物体的勾选即可。但是有没有考虑过,游戏中模型要实现被击穿心脏或被子弹打得千疮百孔,这时就需要透明测试(总不可能单独分出这些Mesh吧),所以存在即合理。
  • 透明度混合顺序渲染即可？
    很不幸把半透明物体按它们距离摄像机的远近进行排序,然后按照从后往前渲染这些半透明物体,问题还是得不到解决的。接下来看看以下情况。 情况1:循环重叠;情况2：A在B之前,但B遮挡A。这样应该怎么办呢？ 有两种方案：第一种是让透明度变得柔和,使其看不出前后关系;第二种是拆解Mesh,如下图所示。
