UnityShader属性交互

UnityShader属性交互

April 19, 2020 字数 1788 极速模式

前言

看过Hello Shader篇章的知道为了实现炫酷的Shader效果,靠基础的Shader知识完全不够用的,使用属性标签是实现炫酷Shader的必经之路。

1.材质面板交互

材质提供了可以方便调节Unity Shader中参数方式,通过这些参数可以随时调整材质的效果,而这些参数就需要写在Properties语义块中,假设需要显示一个颜色拾取器,从而直接控制模型在屏幕上显示的颜色,为此开始编写Shader来实现以上需求。

Shader "Custom/SimpleShader"
{
    Properties{
        _Color("Color Tint", Color) = (1.0,1.0,1.0,1.0)
    }

    SubShader{
        Pass{
            CGPROGRAM
            //使用vert函数和frag函数  
            #pragma vertex vert  
            #pragma fragment frag  
            #include "UnityCG.cginc"  

            //在CG代码中,需要定义与属性名称和类型都匹配的变量
            fixed4 _Color;
        
            //使用结构体来定义顶点着色器的输入
            struct a2v {
                float4 vertex : POSITION;
                float normal : NORMAL;
                float4 texcoord : TEXCOORD0;
            };

            struct v2f {
                float4 pos : SV_POSITION;
                fixed3 color : COLOR0;
            };
        
            v2f vert(a2v v){
                v2f o;
                o.pos = UnityObjectToClipPos(v.vertex);
                o.color = v.normal * 0.5 + fixed3(0.9, 0.3, 0.5);
                return o;
            }

            fixed4 frag(v2f o) : SV_Target{
                fixed3 c = o.color;
                c *= _Color.rgb;
                return fixed4(c,1.0);
            }

            ENDCG
    	}
    }
    FallBack "Diffuse"
}

首先在Properties语义块中声明一个属性_Color,它的类型是Color,初始化的值(1.0,1.0,1.0,1.0)对应是白色,为了CG代码中可以访问到它,还需要在CG代码片段中提前定义新的变量,这个变量的Pro名称和类型必须与Properties语义块中的属性定义相匹配。

2.ShaderLab属性与CG变量的匹配

以上代码展示了Color属性与CG变量fixed的对应,Properties还有其他的属性,接下来给出对应的表格。

ShaderLab属性类型 CG变量类型
Color、Vector float4、half4、fixed4
Range、Float float、half、fixed
2D sampler2D
Cube samplerCube
3D sampler3D

有时在CG变量钱会有uniform关键字,例如以下代码段:

uniform fixed4 _Color;

uniform关键字是CG中修饰变量和参数的修饰词,它仅仅是用于提供关于该变量的初始值是如何指定和存储相关信息(和其他图像编程接口中的uniform关键字作用不太一样的),在UnityShader中可以省略此关键词。

3.Unity的内置文件和变量

在Unity中编写基本的顶点、片元着色器的过程。顶点、片元着色器的复杂之处在于很多事情需要亲自去实现,比如转换法线方向、处理光照、阴影等。为了避免开发者重复创造轮子,Unity提供了很多内置文件,这些文件包含很多提前定义的函数、变量和宏。包含文件类似于C++中头文件引入的方式。在Unity中文件后缀是.cginc,在编写Shader时可以通过#include把这些文件包含进来,这样就可以直接使用Unity定义好的有用的变量和帮助函数。

#include "UnityCG.cginc"

UnityCG这个文件编译时将自动包含进来,这些文件存在Unity安装目录下的Editor\Data\CGIncludes里,如果安装目录找不到这些.cginc文件可以去Unity官网下载,如果想要学习内置着色器的实现或内置函数的实现时,都可以直接去查看这些文件内部实现。下面给出表格(编译时会自动包含进来的文件)。

文件名 描述
UnityCG.cginc 包含最常用使用的帮助函数、宏、结构体等。
UnityShaderVariables.cginc 包含许多基本的内置全局函数,比如UNITY_MATRIX_MVP矩阵等。
Lighting.cginc 包含各种内置光照模型,编写Surface Shader的会自动包含进来。
HLSL.Support.cginc 包含很多用于跨平台编译的宏定义。
UnityStandardBRDF.cginc 对应不同的BRDF模型实现。
UnityStandardCore.cginc 实现基于物理系统的渲染效果。

UnityCG.cginc是经常接触到的包含文件,下面整理经常用到的结构体和帮助函数表格。

结构体名称 描述 包含变量
appdata_base 可用于顶点着色器的输入 顶点位置、顶点法线、第一组纹理坐标
appdata_tan 可用于顶点着色器的输入 顶点位置、顶点切线、顶点法线、第一组纹理坐标
appdata_full 可用于顶点着色器的输入 顶点位置、顶点切线、顶点法线、四组以上纹理坐标
appdata_img 可用于顶点着色器的输入 顶点位置、第一组纹理坐标
v2f_img 可用于顶点着色器的输出 剪裁空间位置、纹理坐标
函数 描述
float3 WorldSpaceViewDir(float4 v) 输入模型空间中顶点位置,返回世界空间中从该点到摄像机的观察方向
float3 ObjSpaceViewDir(float4 v) 输入模型空间中的顶点位置,返回模型空间中从该点到摄像机的观察方向
float3 WorldSpaceLightDir(float4 v) 仅可用于前向渲染中,输入模型空间中的顶点位置,返回世界空间从该点到光源的光照方向,矢量没有被归一化
float3 ObjSpaceLightDir(float4 v) 仅可用于前向渲染中,输入模型空间中的顶点位置,返回模型空间从该点到光源的光照方向,矢量没有被归一化
float3 UnityObjectToWorldNormal(float3 norm) 把法线方向从模型空间转换到世界空间中
float3 UnityObjectToWorldDir(in float3 norm) 把方向矢量从模型空间转换到世界空间中
float3 UnityWorldToObjectDir(float3 norm) 把法线方向从世界空间转换到模型空间中
