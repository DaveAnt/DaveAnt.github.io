UnityShader基本光照

UnityShader基本光照

April 21, 2020 字数 3411 极速模式

前言

一切物体都受到光照的影响,游戏要模拟出现实世界的光照,自然而然出现很多关于光照的理论和知识,当然理论为了性能考虑并不是完全和现实世界的同样,有些东西在游戏中看上去效果是差不多的,它就可以被应用于场景中。

1.光照基本理论

光线照射到物体上,一些光线被吸收了,另外一些光线被散射到其他方向,最终摄像机可以吸收到光,就是看到的最终影像。散射只改变光线方向,但不改变光线密度和颜色。而吸收只改变光线密度和颜色而不改变光线方向。光线在物体表面经过散射后,有两种方向：一种将散射到物体内部被称为折射或透射;另外一种是散射到外部,这种现象被称为反射。 在游戏中会进入到摄像机大致会有4种光源。

  • 自发光(emissive)
    表面本身发射出的光照强度。如果没有使用全局光照技术,这些自发光表面并不会照亮周围物体,而是本身看上去更亮。
  • 高光反射(specular)
    当光线从光源照射到模型表面,该表面在镜面方向完全反射出的光照强度,它是经验模型并不完全符合现实,可以让模型看上去更加光泽。
  • 漫反射(diffuse)
    当光线从光源照射到模型表面时,该表面会散射出的光照强度。
  • 环境光(ambient)
    其他所有间接的光照,光线通常会在多个物体之间不止一次的反射,最后进入到摄像机。

光照模型可以在顶点(逐顶点光照)、片元(逐像素光照)着色器中计算。在逐像素光照中,会以每个像素为基础得到它的法线,然后继续光照模型计算,这种面片之间对顶点法线继续插值技术被称为Phong着色。与之相对的是逐顶点光照,也被称为高洛德着色,在逐顶点光照中,每个顶点进行光照计算,然后会在渲染图元内部进行线性插值输出最后像素颜色。标准光照模型仅仅是经验模型,它并不完全符合真实世界中的光照现象,有很多重要的光照的物理现象无法表现。但用于易用性、计算速度和得到比较好的效果,因此仍然被广泛使用。

2.Unity中实战

  • Unity下的环境光、自发光
    在Unity场景中环境光可以在Window/Rendering/Lighting Settings下找到关于环境光的设置。如图所示,可以改变环境光的颜色、强度、模式。 而大多数物体时没有自发光的效果,但是要计算自发光的话十分简单,只需要在片元着色器输出最后颜色前,把材质自发光颜色添加到输出颜色上即可。
  • UnityShader漫反射光照模型
    首先给出基本光照模型中漫反射部分的计算方式：
    cdiffuse=(clight.mdiffuse)max(0,nf.I)
    从公式上可以看到计算漫反射要四个参数：入射光线的颜色和强度clight,材质的漫反射系数mdiffuse,表面法线nf以及光源方向I,接下来尝试编写Shader代码：
Shader "Unlit/DiuffuseShader"
{
    Properties
    {
        _Diffuse("Diffuse",Color) = (1,1,1,1)
    }
    SubShader
    {
        Tags { "LightMode"="ForwardBase" }
        LOD 100

        Pass
        {
            CGPROGRAM
            #pragma vertex vert
            #pragma fragment frag
            #include "Lighting.cginc"

            struct a2v
            {
                float4 vertex : POSITION;
                float3 normal : NORMAL;
            };

            struct v2f
            {
                float4 vertex : POSITION;
                float3 color : COLOR;
            };
    
            fixed4 _Diffuse;

            v2f vert (a2v v)
            {
                v2f o;
                o.vertex = UnityObjectToClipPos(v.vertex);

                fixed3 worldNormal = normalize(mul(v.normal,(float3x3)unity_WorldToObject));
                fixed3 worldLight = normalize(_WorldSpaceLightPos0.xyz);
                
                fixed3 ambient = UNITY_LIGHTMODEL_AMBIENT.xyz;
                fixed3 diffuse = _LightColor0.rbg * _Diffuse.rgb * saturate(dot(worldNormal,worldLight));
                o.color = ambient + diffuse;
                
                return o;
            }

            fixed4 frag (v2f i) : SV_Target
            {
                return fixed4(i.color,1.0);
            }
            ENDCG
        }
    }
    FallBack "Diffuse"
}

在结构体a2v中定义了normal变量是为了访问顶点法线,并通过使用NORMAL语义告诉Unity要把模型顶点法线信息存储到normal变量中.为了把顶点着色器计算出的光照颜色传递给片元着色器,在v2f中定义了color变量。回忆一下上面那个公式,计算漫反射需要做的4个参数。
经过前面一系列步骤已经知道材质漫反射颜色_Diffuse以及顶点法线v.normal,剩下的就是光源颜色、强度、方向,Unity提供了内置变量_LightColor0来访问该Pass处理的光源颜色、强度(定义合适的LightMode标签才会获取到合适值),最后光源方向可以通过_WorldSpaceLightPos0.xyz获取(当场景中只有一个平行光才适用,不然得不到正确结果)。
得到模型空间到世界空间的变换矩阵的逆矩阵_World2Object,通过调换它的mul函数中的位置得到和转置矩阵相同的矩阵乘法,由于法线是三维矢量,因此只需要_World2Object前三行前三列即可。将得到的结果归一化之后需要等到点积,为了防止点积结果为负值,通过CG提供的saturate函数,使参数截取到[0,1]的范围内,最后与光源的颜色、强度、材质漫反射颜色相乘得到最终漫反射光照部分。

以上是逐顶点光照实现漫反射的Shader代码,其实改变成逐片元光照也很简单,只需要把世界空间下的法线传递给片元着色器即可,部分代码如下:

    struct a2v
    {
        float4 vertex : POSITION;
        float3 normal : NORMAL;
    };

    struct v2f
    {
        float4 vertex : POSITION;
        float3 worldNormal : TEXCOORD0;
    };

    v2f vert (a2v v)
    {
        v2f o;
        o.vertex = UnityObjectToClipPos(v.vertex);
        o.worldNormal = mul(v.normal,(float3x3)unity_WorldToObject);
        
        return o;
    }

    fixed4 frag (v2f i) : SV_Target
    {
        fixed3 worldNormal = normalize(i.worldNormal);
        fixed3 worldLight = normalize(_WorldSpaceLightPos0.xyz);
        
        fixed3 ambient = UNITY_LIGHTMODEL_AMBIENT.xyz;
        fixed3 diffuse = _LightColor0.rbg * _Diffuse.rgb * saturate(dot(worldNormal,worldLight));
        fixed3 color = ambient + diffuse;
        return fixed4(color,1.0);
    }

完成以后去查看模型背光面会发现是纯黑的,失去了模型细节表现为改善这情况,公式被修改成：
cdiffuse=(clight.mdiffuse)(α.nf.I+β)
一般会把α和β都设置成0.5,以上公式称为半兰伯特光照模型,它不需要进行max判断,是Value公司在开发《半条命》时提出的技术,还是应了那句图形学只要看上去是对的、效果不错的,那它就是可行的。

  • UnityShader高光反射模型
    首先给出基本光照模型中高光反射部分的计算方式：
    cspecular=(clight.mspecular)max (0,v.r)mglass
Shader "Unlit/SpecularShader"
{
    Properties
    {
        _Diffuse("Diffuse",Color) = (1,1,1,1)
        _Specular("Specular",Color) = (1,1,1,1)
        _Gloss("Gloss",Range(8.0,256)) = 20
    }
    SubShader
    {
        Pass
        {
            Tags{ "LightMode"="ForwardBase" }

            CGPROGRAM
            #pragma vertex vert
            #pragma fragment frag

            #include "UnityCG.cginc"
            #include "Lighting.cginc"

            fixed4 _Diffuse;
            fixed4 _Specular;
            float _Gloss;

            struct a2v
            {
                float4 vertex : POSITION;
                float2 normal : NORMAL;
            };

            struct v2f
            {
                float4 pos : SV_POSITION;
                fixed3 color : COLOR;
            };

            v2f vert (a2v v)
            {
                v2f o;
                o.pos = UnityObjectToClipPos(v.vertex);

                fixed3 ambient = UNITY_LIGHTMODEL_AMBIENT;

                fixed3 worldNormal = normalize(mul(v.normal,(float3x3)unity_WorldToObject));
                fixed3 worldLightDir = normalize(_WorldSpaceLightPos0.xyz);

                fixed3 diffuse = _LightColor0.rgb*_Diffuse.rgb*saturate(dot(worldNormal,worldLightDir));
                fixed3 refractDir = normalize(reflect(-worldLightDir,worldNormal));
                fixed3 viewDir = normalize(_WorldSpaceCameraPos.xyz - mul(unity_ObjectToWorld,v.vertex).xyz);
                fixed3 specular = _LightColor0.rgb*_Specular.rgb*pow(saturate(dot(refractDir,viewDir)),_Gloss);

                o.color = ambient + diffuse + specular;

                return o;
            }

            fixed4 frag (v2f i) : SV_Target
            {
                return fixed4(i.color,1.0);
            }
            ENDCG
        }
    }

    FallBack "Specular"
}

部分计算代码和漫反射是完全一致,首先计算入射光线方向关于表面法线的反射方向reflectDir,由于CG的reflect函数的入射方向要求是由光源指向交点处的,需要对worldLightDir取反后再传给reflect函数,然后通过_WorldSpaceCameraPos得到了世界空间中的摄像机位置,再把顶点位置从模型空间变换到世界空间下,再通过和_WorldSpaceCameraPos相减即可得到世界空间下的视角方向。得到四个参数代入公式即可得到高光反射的光照部分。
使用逐顶点光照得到的高光效果有比较大的问题,可以看出高光部分明显不平滑,这是因为高光反射部分计算是非线性的,而在顶点着色器中计算光照再进行插值的过程是线性的,破坏了原有计算的非线性关系,就会出现比较大的视角问题,因此需要使用逐像素的方法来计算高光反射。

接下来就使用逐像素来实现高光反射,只需要略微的调整即可变成逐像素渲染。

	struct v2f
    {
        float4 pos : SV_POSITION;
        float3 worldNormal : TEXCOORD0;
        float3 worldPos : TEXCOORD1;
    };

    v2f vert (a2v v)
    {
        v2f o;
        o.pos = UnityObjectToClipPos(v.vertex);

        o.worldNormal = mul(v.normal,(float3x3)unity_WorldToObject);
        o.worldPos = mul(unity_ObjectToWorld,v.vertex).xyz;

        return o;
    }

    fixed4 frag (v2f i) : SV_Target
    {
        fixed3 ambient = UNITY_LIGHTMODEL_AMBIENT;

        fixed3 worldNormal = normalize(i.worldNormal);
        fixed3 worldLightDir = normalize(_WorldSpaceLightPos0.xyz);

        fixed3 diffuse = _LightColor0.rgb*_Diffuse.rgb*saturate(dot(worldNormal,worldLightDir));
        fixed3 refractDir = normalize(reflect(-worldLightDir,worldNormal));
        fixed3 viewDir = normalize(_WorldSpaceCameraPos.xyz - i.worldPos.xyz);
        fixed3 specular = _LightColor0.rgb*_Specular.rgb*pow(saturate(dot(refractDir,viewDir)),_Gloss);
        
        return fixed4(ambient + diffuse + specular,1.0);
    }

另一个高光反射模型是Blinn,具体计算公式: cspecular=(clight.mspecular)max(0,n.h)mglass
只需要修改片元着色器的部分代码即可。

fixed4 frag (v2f i) : SV_Target
{
    fixed3 ambient = UNITY_LIGHTMODEL_AMBIENT.xyz;

    fixed3 worldNormal = normalize(i.worldNormal);
    fixed3 worldLightDir = normalize(_WorldSpaceLightPos0.xyz);

    fixed3 diffuse = _LightColor0.rgb*_Diffuse.rgb*saturate(dot(worldNormal,worldLightDir));

    fixed3 viewDir = normalize(_WorldSpaceCameraPos.xyz - i.worldPos.xyz);
    fixed3 halfDir = normalize(worldLightDir + viewDir);
    fixed3 specular = _LightColor0.rgb*_Specular.rgb*pow(saturate(dot(worldNormal,halfDir)),_Gloss);
    
    return fixed4(ambient + diffuse + specular,1.0);
}

3.关于光照的内置函数

可以看到上面有很多类似的代码,比如normalize(_WorldSpaceLightPos0.xyz)得到光源的方向(只适合平行光),如果有更复杂的光照类型,以上代码是处理不了的。于是UnityCG.cginc中一些非常有用的帮助函数,以下列出表格是经常使用的一些内置函数。

函数名 描述
float3 WorldSpaceViewDir(float4 v) 输入模型空间中的顶点位置,返回世界空间中从该点到摄像机观察方向,内部实现使用了UnityWorldSpaceViewDir函数。
float3 UnityWorldSpaceViewDir(float4 v) 输入世界空间中的顶点位置,返回世界空间中从该点到摄像机的观察方向。
float3 ObjSpaceViewDir(float4 v) 输入模型空间中的顶点位置,返回模型空间中该点到摄像机的观察方向。
float3 WorldSpaceLightDir(float4 v) 仅可用于向前渲染中。输入模型空间中的顶点位置,返回世界空间中从该点到光源的光照方向。内部实现使用UnityWorldSpaceLightDir函数,没有被归一化。
float3 UnityWorldSpaceLightDir(float4 v) 仅可用于向前渲染中。输入世界空间中的顶点位置,返回世界空间中从该点到光源的光照方向。内部实现使用UnityWorldSpaceLightDir函数,没有被归一化。
float3 ObjectSpaceLightDir(float4 v) 仅可用于向前渲染中。输入模型空间中的顶点位置,返回模型空间中从该点到光源的光照方向。内部实现使用UnityWorldSpaceLightDir函数,没有被归一化。
float3 UnityObjectToWorldNormal(float3 norm) 把法线方向从模型空间转换世界空间中
float3 UnityObjectToWorldDir(in float3 norm) 把方向矢量从模型空间转换世界空间中
float3 UnityObjectToObjectDir(float3 norm) 把方向矢量从模型空间转换模型空间中

接下来就直接使用内置函数实现Blinn光照。

v2f vert(a2v v)
{
    v2f o;
    ...
    o.worldNormal = UnityObjectToWorldNormal(v.normal);
    ...
    return o;
}

fixed4 frag(v2f i):SV_Target
{
    ...
    fixed3 worldNormal = normalize(i.worldNormal);
    fixed3 worldLightDir = normalize(UnityWorldSpaceLightDir(i.worldPos));
    fixed3 worldLightDir = normalize(UnityWorldSpaceLightDir(i.worldPos));
    ...
}
