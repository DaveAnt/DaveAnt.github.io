命令模式

March 18, 2020 字数 2020 极速模式

前言

命令模式可以实现操作的前进和回滚,经常使用P图软件的用户一定知道软件提供了对操作前进和回滚的功能,实际上可以记录每个操作节点可以实现此效果。

1.实现操作节点模块

其实挺多关于设计方面的软件都需要后退和前进的功能,要实现这个功能就需要把每个动作都存下来,这个时候命令模式就出现了,将对软件每个操作都可封装成对象然后保存到列表里,就可以实现操作的可控,就像幻灯片可以步步前进或者后退,接下来先实现一下代码:

    public interface IOrder
    {
        void Done();
        void Undo();
    }
 
    public struct Point
    {
        public int x;
        public int y;
    }
 
    public struct Size
    {
        public uint width;
        public uint height;
    }
 
    public enum Color
    {
        white,
        black,
        red,
        green,
        blue,
    }
 
    public class OrderWorks
    {
        private static int backPos = 0;
        private static int forwardPos = 0;
        static List<IOrder> executeOrders = new List<IOrder>();
        static List<IOrder> saveOrders = new List<IOrder>();
 
        public static void Forward()
        {
            if (forwardPos < saveOrders.Count)
            {
                saveOrders[forwardPos].Done();
                forwardPos = forwardPos + 1;
                backPos = forwardPos - 1;
            }
 
        }
 
        public static void Back()
        {
            if (backPos >= 0)
            {
                saveOrders[backPos].Undo();
                backPos = backPos - 1;
                forwardPos = backPos + 1;
            }
        }
 
        public static void Excute(IOrder order)
        {
            forwardPos = forwardPos + 1;
            backPos = forwardPos - 1;
            executeOrders.Add(order);
            saveOrders.Add(order);
        }
 
        public static void Update()
        {
            while (true)
            {
                foreach (IOrder order in executeOrders)
                {
                    order.Done();
                }
                executeOrders.Clear();
                Thread.Sleep(100);
            }
        }
    }
 
    public class EraserOrder:IOrder
    {
        private Size size;
        private Point point;
        private int layer;
        private readonly EraserModule eraserModule;
        public EraserOrder(EraserModule eraserModule)
        {
            this.eraserModule = eraserModule;
            this.size = eraserModule.size;
            this.point = eraserModule.point;
            this.layer = eraserModule.layer;
        }
 
        public void Done()
        {
            eraserModule.DoEraser();
        }
 
        public void Undo()
        {
            eraserModule.UnDoEraser();
        }
    }
 
    public class PaintOrder : IOrder
    {
        private Color color;

实例代码就相当于就实现了操作前进和回滚,不能直接调用对象里面代码,因为需要记录操作,接下来是具体的调用方式:

OrderWorks.Excute(new BrushOrder(BrushModule.instance));

调用OrderWorks.Excute其实没有执行具体的操作,只是添加到列表里,实际调用存在OrderWorks.Update,100毫秒会去刷新一次执行操作,因为要记录操作方式,这样每次都是需要new一次把每次操作的参数保存下来。唯一不方便的就是每次有新的操作方面的功能,就要多写一个把操作封装成对象的类。

2.以上代码隐藏BUG

当完善以上代码去测试的时候,发现调用OrderWorks.Excute(new BrushOrder(BrushModule.instance))时,调用时存进去的BrushModule.instance.size是(3,3),然后修改一下Size大小,调用回滚操作的函数,发现执行的代码参数就是当前的,而不是(3,3),因为DoBrush就是直接调用自身的参数的,与new BrushOrder()下的参数毫无关联,所以接下来我们需要把这个bug修改一下:

总结

1.命令模式优缺点

优点:1、降低了系统耦合度。 2、将一个操作封装成一个对象,实现了操作可控。
缺点:使用命令模式会导致系出现过多的具体命令类。

2.命令模式使用场景

1.存在操作节点回滚和前进的系统
2.GUI 中每个按钮都是一条命令
